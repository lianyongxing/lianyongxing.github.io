# 安装Mongodb_exporter

真正将mongodbmongodbmongodb与prometheusprometheusprometheus连接的是mongodb_exportermongodb\_exportermongodb_exporter，这个东西就像一个桥梁，用户在mongodbmongodbmongodb中进行了什么操作，会先告诉mongodb_exportermongodb\_exportermongodb_exporter，之后mongodb_exportermongodb\_exportermongodb_exporter再告诉prometheusprometheusprometheus。

为什么这么麻烦搞一个中间人？是因为GrafanaGrafanaGrafana的默认数据源不包括mongodbmongodbmongodb，所以需要将prometheusprometheusprometheus作为数据源，用mongodbmongodbmongodb作为prometheusprometheusprometheus的输入端。而prometheusprometheusprometheus与mongodbmongodbmongodb之间的连接需要mongodb_exportermongodb\_exportermongodb_exporter。

同样在dockerdockerdocker中安装mongodb_exportermongodb\_exportermongodb_exporter

从dockerhubdocker \: hubdockerhub里 pullpullpull 下来 mongodb_exportermongodb\_exportermongodb_exporter的镜像

docker pull prom/mongodb_exporter

之后运行它（在这里我进行了一个端口映射，为了跟我前面配置的PrometheusPrometheusPrometheus监听的端口对应，因此设置了端口为9001）

docker run -d --name exporter -p 9001:9104 prom/mongodb_exporter --mongodb.uri mongodb://root:root@192.168.19.178:27017

这时mongodb_exportermongodb\_exportermongodb_exporter就配置完了，正式将mongodbmongodbmongodb与prometheusprometheusprometheus连接起来。

现在进入prometheusprometheusprometheus的UIUIUI界面内，点击页面中的status→Targetsstatus \rightarrow Targetsstatus→Targets可以看到StatusStatusStatus已经由DownDownDown改为了UpUpUp

到此为止

mongodb→mongodbexporter→prometheusmongodb \rightarrow mongodbexporter \rightarrow prometheus mongodb→mongodbexporter→prometheus

已经调通。
